The ELIPPSS Crew

This project is the result of an extensive group collaboration. The seven teams depended on one another by providing data and metadata, maintaing delivery deadlines and helping eachother out in the field. The following section provides short insight into the individial team's work.

Project Management

The project management group had the responsibility to coordinate, evaluate, monitor and provide assistance to the other groups. This included the initial task of evaluating the groups' needs, time management to fulfill the course deadlines and the final report submission. The organization of communication platforms like Trello, Slack and Google Drive helped ensure deadlines were maintained and communication among the different groups ran smoothly

Sensor Platforms

Sensor and platform group was responsible for all the tasks required for the collection of field data using UAV. The main tasks included the selection of appropriate sensors and their integration with the chosen platform, flight planning, UAV setup, sensor setup, establishment of control points, etc. Besides this there were some intermediate tasks such as 3D printing to mount the sensors to UAV gimbal, data management and data distribution.

LiDAR

LiDAR team's main goal was to develop a strategy to deal with the available equipment to generate highly accurate 3D models (DEM and DSM) for the project’s study area. The focus lay mostly on an approach to work with a low-cost LiDAR scanner (provided by Sweep) on the outdoor environment. Since the scanner was designed for indoor data capturing, its capabilities were pushed forward to generate a solution for integrating it in the landscape monitoring process. The final products are: merged and classified LAS point clouds and georeferenced DEM and DSM. The products were passed on to the ESRI cloud group for processing and storing.

SenseBox

The senseBox group takes its name from the SenseBox project developed by the Institute for Geoinformatics in the University of Münster. “The senseBox is a do-it-yourself kit for stationary and mobile sensor stations”. The SenseBox group used the senseBox to build different kind of sensors to monitor the environmental quality of the river Aa study area. The sensors built were: a stationary weather logger, a stationary water level logger, a mobile air quality logger mounted on a drone, and a mobile pH and water temperature logger.

Image Analysis

Provision of meaningful spatial products and services is integral in process of monitoring the conservation and renaturation of the environment. This project group aimed to analyze cost efficient and high spatial resolution images acquired by using unmanned aerial systems to understand the process of renaturation of Aa river near Münster, Germany. Among the generated products were: DSM, Orthorectified multiespectral image, orthophoto, NDVI, cloud points, 3d-mesh and image classification. This set of products was shared with Esri and Web group. ESRI cloud

The ESRI Cloud Group's main goal was to create web applications to visualize 2D and 3D data using the ESRI Commercial Platform’s products, namely ArcGIS Pro, ArcMap, Collector for ArcGIS and ArcGIS Online. Tasks were closely related to the data acquirement by other groups using a geodatabase, producing vector and raster data visualizations (both 2D and 3D), sharing the outputs in the Web Portal, sharing feature classes with field collaborators to collect and store data using the app Collector for ArcGIS, giving data access to the Webteam according to the WMS protocol, and designing and customizing the 2D and 3D ArcGIS web applications.

Web Portal

The purpose of the Web Portal group was to develop this web-platform in which the obtained data of the project and related information is presented to the public. Allowing the exploratory visualization, interaction with several products. There were no restrictions concerning the technologies used and the features implemented as long as the generated data was displayed and accessible in a user friendly, efficient way. The web technologies included in the portal development were mainly Javascript, HTML, CSS, AngularJS, Leaflet. The data displayed on the map are mostly WMS layers stored in the ESRI cloud, which was managed by the ESRI cloud group.
